Calculating a Sort Algorithm Time complexity

03 March 2015

Time the time complexity of an algorithm quantifies the amount of time taken by an algorithm to run as a function of the length of the string representing the input.

-Wikipedia

What

Time complexity is used to measure how efficient your algorithm really is. This is done by counting the number of times your algorithm must go through every element in your list until you are able to achieve whatever you are trying to achieve.

Example:

If you want to find the largest number in this list. 

unsortedList = [3,43,2]

If we compare every element in the list then the time complexity of that algorithm would be O(n). O = the notation used to convey time complexity.

Why

The reason why we evaluate time complexity is because there are multiple ways to achieve the same thing. Some methods are more efficient (meaning faster run time and/or less CPU demand). The time complexity of an algorithm might seem irrelevant until you start getting huge sets of data. Then that performance can quickly matter from a few seconds to hours and days of a difference.
